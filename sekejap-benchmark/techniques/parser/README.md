# SQL Parser Benchmark

This suite isolates Sekejap SQL frontend cost from engine execution cost.

It benchmarks the current in-repo parser in [`src/sql/parser.rs`](C:/Users/Insan/AndroidStudioProjects/SekejapDB/src/sql/parser.rs), with focus on:

- graph-first `SELECT ... TRAVERSE ...`
- constrained graph `SELECT` with exact-time and text predicates
- multi-row `INSERT ... VALUES ...`

The goal is to answer:

1. how much time is spent parsing SQL before atomic execution begins
2. whether large `VALUES` batches are already pathological at parse time
3. whether the next step should be:
   - switch general SQL parsing to a more robust library
   - add a specialized fast-path write parser
   - or both

The result file is generated by:

```powershell
cargo run -p sekejap-benchmark-rs --bin parser_benchmark
```
